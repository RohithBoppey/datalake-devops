Q. Boto3 has things like choose the location like: ap-south-1 etc right? How can we use the same Boto3 client with Minio then? 

A. When you connect boto3 to MinIO, you're essentially telling it:

> "Don't talk to AWS at all. Talk to this specific URL instead. And don't worry about regions --- just send the requests there directly."

You do that with two parameters:

-   `endpoint_url` --- bypasses AWS entirely, points boto3 at your MinIO URL
-   `aws_access_key_id` / `aws_secret_access_key` --- these aren't "AWS credentials" anymore, they're just MinIO's username and password, but boto3 uses the same fields because the auth mechanism is identical
