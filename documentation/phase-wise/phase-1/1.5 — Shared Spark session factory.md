### 2 ways to connect to a spark session: 

#### 1. via exec-ing into the container
```
docker exec -it dataeng_devops-spark-1 bash 
```

```
/opt/spark/bin/pyspark \
  --packages io.delta:delta-spark_2.12:3.1.0 \
  --conf "spark.jars.ivy=/tmp/.ivy2" \
  --conf "spark.sql.extensions=io.delta.sql.DeltaSparkSessionExtension" \
  --conf "spark.sql.catalog.spark_catalog=org.apache.spark.sql.delta.catalog.DeltaCatalog"
```
- we generally use this mode to run a spark container, just to run the queries only, just a temporary session to read a dataframe, etc. 

#### 2. via the host system while spark is running as container
- for this, we are eseentially using the pyspark of the host, in which we give the spark flags during runtime
- can also give the S3 credentials for this to test, end to end

```
pyspark \
  --packages io.delta:delta-spark_2.12:3.1.0,org.apache.hadoop:hadoop-aws:3.3.4 \
  --conf "spark.sql.extensions=io.delta.sql.DeltaSparkSessionExtension" \
  --conf "spark.sql.catalog.spark_catalog=org.apache.spark.sql.delta.catalog.DeltaCatalog" \
  --conf "spark.hadoop.fs.s3a.endpoint=http://localhost:9000" \
  --conf "spark.hadoop.fs.s3a.access.key=minio" \
  --conf "spark.hadoop.fs.s3a.secret.key=minio123" \
  --conf "spark.hadoop.fs.s3a.path.style.access=true" \
  --conf "spark.hadoop.fs.s3a.impl=org.apache.hadoop.fs.s3a.S3AFileSystem"
```

now "spark session" is the beginning of all spark, from terminal the pyspark actually creates the session for it

but if I want to run a py script in which I'm interacting via the spark session, then how?? 

- In a Python script, nobody creates it for you. You have to create it yourself, in code, with all those same configs you were passing as `--conf` flags.
- That is exactly what `spark_session.py` does — it's a function that creates and returns a configured `SparkSession` so that the rest of your code can use it.

```
delta_client.py
      │
      │  calls
      ▼
spark_session.py  ←  creates a SparkSession with the right JARs + MinIO config
      │
      │  returns a SparkSession object
      ▼
delta_client.py uses that session to talk to Delta + MinIO
```


```
Your pipeline script (e.g. seed_orders.py)
      │
      │  calls get_spark("delta")
      ▼
spark_session.py
      │  creates SparkSession with:
      │  - Delta JAR loaded
      │  - Delta SQL extensions enabled
      │  - MinIO endpoint configured
      │  returns spark object
      ▼
delta_client.py receives spark object
      │
      │  uses spark + delta-spark Python package
      │  to read/write/delete/history
      ▼
Delta JAR (Java, loaded by SparkSession)
      │
      ▼
Parquet files + _delta_log/ on MinIO
```
