so far, Spark now has worker & master containers and in both of them currently, only Java and Spark, Python are present in the same base image, but have no understanding on how to connect with Delta.  

Ex py script that gets ran in the container: 
`from delta import DeltaTable`

but that delta package is not present in the python of the Spark container, so we have to install it while setup only! 

and even if this is installed, Spark still has no idea how to interact with the Delta engine to read / write, so it requires Jars in it

Think of it Like a Car
A car has two things:
- A steering wheel — what you (the driver) touch and interact with
An engine — what actually does the work under the hood
- You never touch the engine directly. You turn the steering wheel, and the steering wheel talks to the engine.     

Delta Lake works exactly the same way:
- `delta-spark` Python package = the steering wheel (what your code touches)
- `Delta JAR` = the engine (what actually reads/writes Delta files)

```
Your delta_client.py
        │
        │  calls
        ▼
delta-spark Python package        ← you installed this via pip
        │
        │  translates your Python calls into
        │  Spark SQL / DataFrame operations
        ▼
Apache Spark (already in container)
        │
        │  hands off Delta-specific operations to
        ▼
Delta JAR (Java, loaded by Spark)  ← Spark downloads this via spark.jars.packages
        │
        │  reads and writes
        ▼
Parquet files + _delta_log/ on MinIO
```
